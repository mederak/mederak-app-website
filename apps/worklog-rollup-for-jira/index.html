How the panel works.

The app stays inside Jira and reads standard Jira time tracking fields.

1

Open an issue

The panel reads the current Jira issue context.

2

Find descendants

Jira issues are retrieved through the standard parent relationship.

3

Add time

The app totals Jira timespent and timetracking values.

4

Review details

Users see total MH/MD and the issues included in the calculation.

Designed for the Jira issue view.

The panel focuses on the numbers users need while keeping the issue context visible.

Data model

Uses Jira work data and Forge Storage cache.

The app reads issue hierarchy and time tracking fields from Jira Cloud. It uses Forge Storage for short-lived calculation cache.

✓ Standard Jira fields: timespent and timetracking.timeSpentSeconds. ✓ Jira parent relationship for descendants. ✓ No Tempo data, rates, costs or billing calculations.
