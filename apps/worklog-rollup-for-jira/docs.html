User guide

Documentation

Worlog Rollup for Jira shows logged time for the current issue and descendant issues directly in the Jira Cloud issue view.

Using the panel

  1. Open a Jira Cloud issue.
  2. Open the Worlog Rollup issue panel.
  3. Review the total logged time in MH and MD.
  4. Use the details table to see which issues are included in the calculation.
  5. Select Refresh after recent worklog or hierarchy changes.

What is included

The app reads logged time from the current issue and every descendant issue found through Jira's standard parent relationship. This includes deeply nested children, such as sub-tasks under Stories that belong to an Epic. The app uses Jira time tracking fields such as timespent and timetracking.timeSpentSeconds.

Issue details table

The details table lists included issue keys, summaries and time values. It is intended to make the rollup transparent, so users can see which Jira issues contributed to the total.

Units

Time is shown in MH and MD. In the product model, 1 MD equals 8 MH.

Languages and themes

The app supports English, Spanish, French and Polish. The panel is designed for Jira light mode and dark mode.

Refresh and cache

The app uses short-lived Forge Storage cache to reduce repeated calculations during normal page refreshes. The Refresh action requests a fresh calculation.

Large hierarchies

For very large issue trees, the app can stop at safety limits and show a truncated result. The summary then reflects the retrieved data.

What is not included

The app does not calculate costs, rates, billing amounts, invoices or Tempo data. It is focused on Jira worklog time visibility.
