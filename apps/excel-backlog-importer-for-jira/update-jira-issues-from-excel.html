Repeat imports

Update existing Jira issues from Excel

When estimates, descriptions, priorities or labels change after review, a repeat Excel upload can update matching Jira issues instead of creating duplicate work.

When it makes sense

Use update workflows after client review, technical analysis, sprint planning or scope refinement. It is useful when the spreadsheet remains the source of reviewed planning data for a period of time.

Identifying existing issues

Repeat imports need a stable identity. Depending on the import setup, this may be a Jira issue key or a stable external row identity such as External ID, Requirement ID or Backlog ID.

Validation before changes

Before updating Jira, validate mapped fields, required values and duplicate status. This reduces the risk of pushing incorrect spreadsheet values into existing issues.

Preview and report

A preview step helps users review what will be updated. After the run, an import report supports review and troubleshooting.
