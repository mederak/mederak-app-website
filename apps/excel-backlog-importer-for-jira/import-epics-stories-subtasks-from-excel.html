Structured backlog imports

Import Epics, Stories and Sub-tasks from Excel to Jira

Structured Jira imports are harder than flat issue creation. Excel Backlog Importer is designed for structured backlog imports such as Epic to Story to Sub-task workflows where Jira configuration and product rollout support the hierarchy.

The hierarchy problem

Backlogs often arrive as grouped spreadsheet rows: epics, stories and implementation tasks in one workbook. Manual import can lose context when parent names, issue types and relationships are not validated before Jira changes.

Example spreadsheet columns

ColumnPurpose
Epic NameName used to group related stories.
Story SummarySummary for the story-level Jira issue.
Sub-task SummarySummary for task-level rows where supported.
Issue TypeJira issue type to create or update.
DescriptionRequirement or implementation detail.
EstimateTime estimate or effort value supported by the target field.
AssigneeUser reference where Jira lookup and permissions allow it.

How mapping helps

Mapping lets you connect each spreadsheet column to the Jira field it should populate. For hierarchy, the important part is using stable parent or grouping columns so the import flow can validate relationships before issue creation.

Risks to validate

  • Missing issue type or summary.
  • Parent names that do not match expected rows.
  • Unsupported parent-child relationships in the Jira project.
  • Invalid users, estimates, priorities, components or custom field values.
