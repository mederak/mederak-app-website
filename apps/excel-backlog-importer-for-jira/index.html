Workflow

The real eight-screen import flow

These are the actual app screens: start, upload, worksheet selection, column recognition, mapping, hierarchy and duplicate controls, validation, and report.

Open the app in Jira Cloud and confirm that nothing is created until validation and import confirmation.

Drop a modern `.xlsx` workbook or choose it from disk. CSV import is intentionally outside this flow.

Select the sheet, header row and saved setup when the workbook is recognized.

Review detected headers and auto-map common names before detailed field mapping.

Choose the default issue type, optional parent and Jira field mappings for the selected project.

Use the current hierarchy information screen and configure duplicate detection for repeated workbook imports.

Review row-level messages, select valid rows and import only what passed validation.

See created, updated, skipped and failed rows, open Jira issue keys and export CSV or JSON.

Security and data handling

Designed for Jira Cloud and Atlassian Forge.

The app is designed as an Atlassian Forge app. It avoids storing full Excel files and stores only operational metadata needed for duplicate detection, saved mapping setup and reporting.

✓ Works with Jira Cloud and Jira REST APIs through Forge. ✓ No separate vendor-hosted product backend for backlog data is described in current security documentation. ✓ Read the security page, privacy policy and support page.

FAQ

Short answers for teams evaluating Excel to Jira imports.

Can Jira import Excel files directly?

Jira's native import workflow is commonly CSV-based. This app is designed for XLSX files so teams do not have to rebuild Excel backlogs as CSV first.

Why use this app instead of Jira CSV import?

Use it when your team receives recurring Excel backlogs and needs guided mapping, duplicate handling, validation and reports around that workflow.

Can I import epics, stories and sub-tasks?

The current captured flow presents hierarchy options as informational. Use selected defaults and optional parent setup for the import flow shown on this page.

Can it handle repeated Excel uploads?

Yes. The flow includes duplicate handling and record identity settings for repeated workbook imports.

Does it support custom fields?

The app is designed to map supported Jira fields and supported custom field types. Exact support depends on Jira field configuration.

Can I save mappings for future imports?

Saved setup can help reuse worksheet, header row and mapping choices for repeated imports.

Can I preview the import before creating issues?

Yes. Preview and validation are central to the controlled import flow.

Is it available for Jira Cloud?

Yes. The product is designed for Jira Cloud and Atlassian Forge.
