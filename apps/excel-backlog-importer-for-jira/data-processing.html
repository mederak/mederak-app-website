Data Processing Summary

This page summarizes the data categories processed by Excel Backlog Importer for Jira.

Purpose of processing

To let authorized Jira users import Excel backlog rows into existing Jira projects, update matching Jira issues, validate data before import, save mapping setups, detect duplicates and generate reports.

Categories of data

  • Jira project metadata, issue type metadata and field metadata.
  • Excel row values selected by users for import.
  • Import metadata: file name, sheet name, row identity, fingerprints, import date, importing user and issue keys.
  • Optional user references used for Jira user picker fields.

Storage locations

Forge Storage may store saved setups, import history and duplicate identity maps. Jira issue properties may store import metadata on created or updated issues.

External transmission

The app is designed without a separate vendor-hosted product backend. Backlog data is not sent to servers operated by the app creator. Processing happens through Atlassian Forge and Jira Cloud according to the app scopes approved during installation.

Data minimization

The app is designed not to store full Excel files. Only structured metadata required for product behavior is stored.

Deletion

Customers can uninstall the app and may request deletion of app-stored records by contacting support.jira@mederak.pl.
