User guide

Documentation

A practical guide for preparing Excel backlog files and importing them into Jira Cloud with controlled mapping, validation and reporting.

Overview

Excel Backlog Importer for Jira helps teams create or update Jira issues from structured XLSX files. It is designed for project managers, analysts, Scrum Masters and Jira admins who receive backlog data in spreadsheets.

Before you start

  • Confirm the target Jira project and issue types.
  • Check that you have Jira permissions to create or update issues.
  • Prepare a workbook where each row represents a work item or hierarchy row.
  • Use stable identifiers if you plan to update existing Jira issues later.

Preparing your Excel file

Use clear column names, one header row and consistent values. Avoid merged cells in the import area. Keep issue type, summary and parent/grouping columns explicit.

Supported file format

The product is built around Excel .xlsx workbooks. CSV conversion is not required for the core Excel import workflow.

Mapping Excel columns to Jira fields

During import, map spreadsheet headers to Jira fields. Keep mappings focused on the fields you actually need. Saved setup can help repeat imports with the same file structure.

Working with custom fields

The app is designed to support Jira fields and supported custom field types. Exact behavior depends on Jira configuration, field type and permissions.

Importing epics, stories and sub-tasks

Structured hierarchy imports depend on Jira project configuration and product rollout. Use explicit issue type and parent/grouping columns, then validate the relationship before import.

Updating existing issues

For update workflows, use a stable identity such as Jira Issue Key, External ID, Requirement ID or Backlog ID. Repeated uploads can update matching Jira issues instead of creating duplicate work where the import setup supports the match.

Import history

Import metadata and reports help teams review what happened, support duplicate handling and troubleshoot later questions.

Exporting import reports

After an import, export the result report where available. Reports are useful for project records and post-import review.

Troubleshooting

  • Missing summaries usually mean the mapped Summary column is empty for selected rows.
  • Invalid select values usually mean the spreadsheet value does not match Jira project configuration.
  • User lookup issues can depend on Atlassian account visibility and permissions.
  • Hierarchy errors often come from unsupported parent-child relationships in the Jira project.

Limitations

  • Hierarchy behavior depends on Jira project configuration, issue type hierarchy and permissions.
  • Custom field support depends on the field type and Jira configuration.
  • Marketplace trial and pricing depend on the public Atlassian Marketplace listing.

Support

For help, contact support.jira@mederak.pl. Include your Jira site URL, target project, import goal and a short description of the spreadsheet structure. If you request early access, use a company email address.
