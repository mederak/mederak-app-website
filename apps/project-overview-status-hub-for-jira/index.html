The status-reporting problem

Jira has the work. Managers still have to rebuild the story.

Project status usually lives across boards, issues, comments, due dates, links, owners and separate documents. Project Overview & Status Hub turns that activity into a manager-friendly first draft, then lets humans confirm what should become the official overview.

  • Reduce manual status-report preparation before reviews and steering meetings.
  • Give new joiners a clear starting point for project purpose, owners and links.
  • Spot stale or missing overviews before the portfolio view becomes outdated.

Security and data handling

Designed for Jira Cloud on Atlassian Forge.

The app reads Jira work and user data it is allowed to access, stores overviews and settings in Forge app storage, and uses Atlassian-hosted Forge LLM capabilities for AI-assisted text.

Forge app storage for generated overviews, access policies, settings and refresh state. Jira permissions matter because generation only sees data available to the generating user. Human review keeps suggested owners, links and summaries from becoming official without confirmation.
FAQ

Questions buyers usually ask

Does it replace Jira reports?

No. It organizes project context and signals into readable overviews and portfolio views. It does not replace detailed Jira reporting or project governance.

Does it change Jira issues?

No. It reads Jira data and stores app-specific overviews. It does not move issues or change issue statuses.

Can sensitive sections be hidden?

Yes. Project administrators can restrict sections such as radar or management notes to selected project roles and users.

What happens without AI?

Core project facts, portfolio tables, dates, links, detector signals and freshness checks still work without AI-written summaries.

Who should manage generation?

Generation and refresh are intended for project administrators, Jira administrators or selected PMO users with portfolio management permission.
